Once reports are generated, the Reports section is where you manage them. This page covers how to navigate your report library, find specific reports quickly, understand report status, and clean up reports you no longer need.

## Navigating the Reports Section

The Reports section has three views to help you find what you need:

### All Reports

The complete list of every report in your workspace. Each row shows:

| Column | What It Means |
|--------|--------------|
| Report Type | Rate Report or Expense Report |
| Report Name | The name entered when the report was generated |
| Date Generated | When the report was created |
| Created By | Which user generated it |
| Expiry Period | Days remaining before the report expires (default: 30 days) |
| Views | How many times the report has been opened |
| Actions | Copy link, Open, Delete |

### Recent Reports

Shows only the most recently generated reports. Useful for quickly accessing something you just created without scrolling through older entries.

### Expiring Soon

Flags reports within a few days of their 30-day expiry. Check this view regularly to avoid losing access to reports you still need.

## Searching for a Report

Use the search bar at the top of the Reports section to find reports by name.

<Tip>
Search works best when you use consistent naming conventions. If all your monthly expense reports follow a format like "Fleet Expenses - January 2026", typing "Fleet Expenses" instantly surfaces all of them in order.
</Tip>

## Opening a Report

Click **Open** next to any report to view it. The report opens in a clean, formatted view ready to read, print, or screenshot.

Each time a report is opened, the view count increases. This is useful for knowing whether a client or team member has actually reviewed a shared report.

## Copying a Report Link

Click **Copy link** next to any report to get a shareable URL. You can paste this into:

- A WhatsApp message to a client
- An email to your accounts team
- A Slack or Teams message internally

Anyone with the link can open the report — no login required. Keep this in mind when sharing sensitive financial information.

<Warning>
Report links are not password-protected. Only share them with people who should have access to that financial data.
</Warning>

## Deleting a Report

<Note>
**🔒 Admin Only** - Only Admin users can delete reports. Staff users cannot remove reports from the system.
</Note>

To delete a report:

<Steps>
  <Step title="Find the report">
    Use the search bar or scroll through the All Reports view to locate the report you want to remove.
  </Step>

  <Step title="Click Delete">
    Select the Delete action next to the report.
  </Step>

  <Step title="Confirm deletion">
    Confirm when prompted. Deletion is permanent and cannot be undone.
  </Step>
</Steps>

<Warning>
Deleting a report removes it and its shareable link permanently. Anyone who had access via the link will no longer be able to open it. If you need the report again, you will have to regenerate it from the Expenses or Incomes section.
</Warning>

## Keeping Your Report Library Tidy

A few habits that keep the Reports section easy to navigate:

1. **Name reports clearly from the start** — you cannot rename a report after it is generated
2. **Delete test or duplicate reports promptly** — they clutter the list and confuse your team
3. **Check Expiring Soon at the start of each month** — regenerate anything you still need before it expires
4. **Use the view count** — if a shared report shows zero views, follow up with the recipient

## Common Issues

<AccordionGroup>
  <Accordion title="A report I need has expired">
    Go back to the section where it was originally generated (Expenses or Incomes), apply the same date range filters, and generate a new report. It will appear in your Reports section immediately.
  </Accordion>

  <Accordion title="I cannot find a report I generated last week">
    Use the search bar and type part of the report name. If it still does not appear, check whether it was deleted by another Admin, or whether it may have expired if it was older than 30 days.
  </Accordion>

  <Accordion title="The report shows 0 views but I sent the link">
    The recipient may not have opened it yet, or the link may have been pasted incorrectly. Try copying the link again and resending it.
  </Accordion>
</AccordionGroup>
